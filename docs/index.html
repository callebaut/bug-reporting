🐞 Bug Reporter

A bookmarklet that opens a bug-logging widget on any page — capture, annotate, copy, paste into Jira / Azure DevOps.

1. Set your hosted URL

The bookmarklet loads bug-reporter.js from wherever you host it. This page is served from GitHub Pages, so the field below is pre-filled with this site's URL. Change it if you host elsewhere.

2. Drag this to your bookmarks bar

Drag the button onto your browser's bookmarks bar (or right-click → bookmark the link). Clicking the bookmark on any page opens the widget.

🐞 Report a bug

3. Use it

  
    
    
  1. Open the page with the bug. Reproduce the bug after clicking the bookmark so console & network errors get captured.
    2. 
    
  2. Click the bookmark → the widget opens bottom-right.
    3. 
    
  3. Hit Select an element, hover, and click the element to screenshot it.
    4. 
    
  4. Annotate with the pen / rectangle / arrow tools.
    5. 
    
  5. Add a title & description, then Copy bug report.
    6. 
    
  6. Paste (Ctrl/Cmd+V) into your Jira / Azure DevOps ticket. The screenshot pastes inline and all the context comes with it.
    7. 
  


  

    What gets collected: annotated screenshot, page URL & title, browser & OS, viewport / screen / DPR, device info, your selected element (selector + HTML + position), console errors & warnings, uncaught JS errors, failed network requests, and — optionally — cookies, localStorage & sessionStorage. Designed so an AI agent can pick up the ticket and fix the bug.
    


    ⚠️ Cookies & storage may contain auth tokens. There's a toggle in the widget — leave it off for tickets shared widely.
